使用Hexo+GitHub搭建个人博客

Environment
  • GitHub
  • Markdown

本文主要讲述如何使用Hexo+GitHub来搭建个人博客，搭建的平台是Ubuntu 16.04。从创建GitHub仓库，到安装git、node.js、npm和hexo，讲述了如何来搭建环境。后面还对如何添加页面、添加博文和预览页面，以及如何将这些内容推送到GitHub上进行了详细的讲述，非常适合新手参考。

前言

做笔记是一件很重要的事情，做笔记可以将自己的想法、思路写下来，方便以后查阅，俗话说好记性不如键盘党，做笔记、写博客也可以让自己学会总结、学会分享。
用Hexo配合GitHub就可以用来搭建博客，GitHub作为服务器这样还省去了租用服务器的费用。我搭建的平台是Ubuntu 16.04，在Windows和Mac上搭建的朋友本篇博客也有参考意义，下面就开始吧。

创建GitHub仓库

首先，你需要注册一个GitHub账号，最好取一个有意义的用户名，如果被占用，可以加上数字来区分。本文中假设用户名为：helloworld。
注册账号的地址： https://github.com/ ，输入用户名、邮箱、密码进行注册。然后需要验证刚刚注册用的邮箱，请打开你的邮箱，查看发送给你的确认邮件，你需要验证邮箱后，后面生成的个人主页才会被接受和发布。
然后，创建GitHub仓库，地址： https://github.com/new ，这个仓库的名字必须和你的账号对应，比如：helloworld.github.io，然后输入基本信息，勾选“Initialize this repository with a README”，点击创建仓库。这个仓库的名称（helloworld.github.io）也是你的个人博客网站的网址。
现在，可以访问自己的博客主页了： https://helloworld.github.io/ 。
但是现在这个页面什么都没有，下面需要使用hexo来组织内容。
这里需要注意下面两点：

  • 生成用户/组织的网站页面，所创建仓库的名称为：username.github.io 或 orgname.github.io ；用户名或组织名必须是你自己的账号/或所属的组织，否则GitHub站点不会帮你生成（build，构建，编译）。
  • GitHub页面网站在互联网上是公开，即使其所属的仓库是私有的。如果你有敏感数据在页面仓库中，你可能需要在发布之前删除他们。
